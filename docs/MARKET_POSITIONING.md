# Market positioning / Рыночное позиционирование

## Market pattern / Рыночный шаблон

Current AI website builders commonly lead with prompt-to-site generation, personalised copy and imagery, templates, visual editing and publishing. Wix describes a conversational prompt-to-business-ready-site workflow with an AI agent plus manual pixel-level editing. Squarespace positions Blueprint AI as a guided design partner combining personalised content and design recommendations. Framer presents AI generation from prompts or briefs followed by detailed refinement on a visual canvas.

Современные AI-конструкторы обычно предлагают генерацию сайта из запроса, персонализированные тексты и изображения, шаблоны, визуальное редактирование и публикацию. Wix продвигает разговорное создание готового бизнес-сайта с AI-агентом и ручным pixel-level редактированием. Squarespace позиционирует Blueprint AI как управляемого дизайн-партнёра с персонализированным содержанием и рекомендациями. Framer предлагает генерацию из запроса или brief с последующей детальной доработкой на визуальном canvas.

## CLARYEL position / Позиция CLARYEL

CLARYEL Web Community should not compete on the generic promise "AI makes a website quickly." That promise is already common. The differentiating position is **continuous voice-first ownership**:

CLARYEL Web Community не должен конкурировать общим обещанием «ИИ быстро делает сайт». Такое обещание уже распространено. Отличительная позиция — **постоянное владение сайтом через голосовое управление**:

- the first website begins with a spoken or written story, not a complex editor;
- logo and visual references travel with the brief;
- after publication, the same conversation controls copy, layout, design, functionality, SEO, accessibility and mobile behaviour;
- the user can ask the AI to evaluate design quality, not only execute literal edits;
- GitHub keeps the work reviewable and portable;
- Cloudflare provides a production delivery path;
- the user chooses ChatGPT or another compatible AI rather than being locked to one bundled model.

- первый сайт начинается с устного или письменного рассказа, а не со сложного редактора;
- логотип и визуальные примеры передаются вместе с заданием;
- после публикации тот же разговор управляет текстами, структурой, дизайном, функциями, SEO, доступностью и мобильным поведением;
- пользователь может попросить ИИ оценить качество дизайна, а не только выполнить буквальную правку;
- GitHub сохраняет проверяемость и переносимость работы;
- Cloudflare обеспечивает production-публикацию;
- пользователь выбирает ChatGPT или другой совместимый ИИ без привязки к одной встроенной модели.

## Messaging hierarchy / Иерархия сообщения

1. Outcome: the website starts with a conversation.
2. Continuing value: every later change can also begin with voice.
3. Design control: intent and expert review without learning design software.
4. Confidence: GitHub review, tests, documentation and Cloudflare deployment.
5. Commercial entry: two active sites per free account holder.
6. Technical architecture only after the business value is understood.

1. Результат: сайт начинается с разговора.
2. Постоянная ценность: каждое следующее изменение также начинается с голоса.
3. Управление дизайном: намерение и экспертная оценка без изучения дизайн-программ.
4. Надёжность: проверка GitHub, тесты, документация и публикация Cloudflare.
5. Коммерческий вход: два активных сайта на владельца бесплатного аккаунта.
6. Техническая архитектура объясняется только после понимания бизнес-ценности.

## Claim discipline / Дисциплина заявлений

The public beta currently prepares voice-derived briefs and change histories for an AI application connected to GitHub. It must not claim that the hosted website itself autonomously edits arbitrary repositories without user review. Future direct orchestration must be labelled planned until implemented and tested.

Публичная beta-версия сейчас готовит задания и историю изменений, полученные голосом, для приложения ИИ, подключённого к GitHub. Нельзя заявлять, что размещённый сайт самовольно изменяет любые репозитории без проверки пользователя. Будущая прямая оркестрация обозначается как планируемая до реализации и тестирования.
